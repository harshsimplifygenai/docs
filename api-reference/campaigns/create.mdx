title: "POST /campaigns — Create campaign"
sidebarTitle: "Create campaign"
description: "Create an outbound call campaign with a contact list, retry logic, and optional scheduling. Campaigns stay in draft status until you explicitly resume them."
---

A campaign lets you place outbound calls to a list of contacts using a single API call. You provide the contact list, the agent and phone number to use, and optional retry and scheduling parameters. Campaigns are created in `draft` status and do not start calling until you call the [resume action](/api-reference/campaigns/manage). You can also schedule a campaign to start automatically at a future time.

<Note>
  The Campaigns API uses a different base URL from the core API:
  `https://api.verbeo.ai/campaigns`
</Note>

## Endpoint

```
POST https://api.verbeo.ai/campaigns/create
```

## Request body

<ParamField body="campaign_name" type="string" required>
  Display name for the campaign. Visible in the dashboard.
</ParamField>

<ParamField body="phone_number" type="string">
  Outbound phone number in E.164 format (e.g., `"+12629779611"`). Either `phone_number` or `phone_number_id` is required.
</ParamField>

<ParamField body="phone_number_id" type="string">
  UUID of the phone number in your workspace. Either `phone_number` or `phone_number_id` is required.
</ParamField>

<ParamField body="agent_id" type="string" required>
  UUID of the agent that will handle calls in this campaign.
</ParamField>

<ParamField body="contacts" type="object[]" required>
  Array of contacts to call. Must contain at least one contact. There is no upper limit on the number of contacts.

  <Expandable title="contact object properties">
    <ParamField body="phone_number" type="string" required>
      Contact's phone number in E.164 format.
    </ParamField>
    <ParamField body="contact_name" type="string">
      Contact's display name. Available to the agent as `{{contact_name}}` if referenced in the system prompt.
    </ParamField>
    <ParamField body="custom_data" type="object">
      Any additional key-value pairs for this contact. All fields are injected as dynamic variables during the call and can be referenced in the agent's system prompt as `{{key_name}}`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="max_retries" type="number" default="0">
  Number of additional attempts to make if the initial call does not connect. Range: `0` to `6`. A value of `2` means up to 3 total attempts per contact (1 initial + 2 retries).

  Retries are triggered by: no answer, voicemail detected, busy signal, or dial failure. A contact that answers and then hangs up is considered successful and will not be retried.
</ParamField>

<ParamField body="retry_intervals" type="number[]">
  Array of wait times in minutes between retry attempts. The first value is the delay before the first retry, the second before the second retry, and so on. If the array is shorter than `max_retries`, the last interval is reused for remaining retries.

  Example: `[5, 15, 60]` waits 5 minutes, then 15 minutes, then 60 minutes between attempts.
</ParamField>

<ParamField body="scheduled_at" type="string">
  ISO 8601 UTC timestamp to automatically start the campaign. If provided, the campaign status becomes `scheduled` and calling begins at the specified time. If omitted, the campaign must be started manually via the [resume action](/api-reference/campaigns/manage).

  Example: `"2026-02-15T09:00:00Z"`. The scheduler checks every minute for due campaigns.
</ParamField>

## Response

A successful request returns `201 Created`.

<ResponseField name="success" type="boolean" required>
  `true` when the campaign was created successfully.
</ResponseField>

<ResponseField name="data" type="object" required>
  <Expandable title="data properties">
    <ResponseField name="campaign_id" type="string" required>
      UUID of the created campaign. Store this — you need it to manage the campaign.
    </ResponseField>
    <ResponseField name="campaign_name" type="string" required>
      Display name of the campaign.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Initial status: `"draft"` (no `scheduled_at` provided) or `"scheduled"` (`scheduled_at` was provided).
    </ResponseField>
    <ResponseField name="total_contacts" type="number" required>
      Number of contacts in this campaign.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the campaign was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string" required>
  Human-readable confirmation.
</ResponseField>

## Campaign status lifecycle

After creation, a campaign moves through these statuses:

| Status | Description |
|--------|-------------|
| `draft` | Created but not started. Call the resume action to begin. |
| `scheduled` | Will start automatically at `scheduled_at`. |
| `in-progress` | Actively calling contacts. |
| `paused` | Temporarily stopped. Resume to continue. |
| `completed` | All contacts have been processed. |
| `cancelled` | Archived. All pending contacts were cancelled. |

## Contact status values

Each contact within a campaign has its own status:

| Status | Description |
|--------|-------------|
| `pending` | Waiting to be called |
| `calling` | Call currently in progress |
| `completed` | Successfully reached |
| `failed` | Failed after all retry attempts |
| `retry_scheduled` | Waiting for next retry attempt |
| `no-answer` | No answer received |
| `busy` | Line was busy |
| `cancelled` | Contact cancelled (campaign was archived) |

## Using custom data in your agent

Reference `custom_data` fields in your agent's system prompt using `{{key_name}}` syntax:

```
You are calling {{contact_name}} from {{company}}.
They are interested in {{product_interest}}.
Be prepared to discuss our enterprise pricing.
```

The agent receives the resolved values at call time, enabling fully personalized conversations from a single campaign.

## Error responses

| Code | Status | Description |
|------|--------|-------------|
| `UNAUTHORIZED` | 401 | Invalid or missing API key |
| `MISSING_FIELD` | 400 | Required field not provided |
| `INVALID_PHONE_FORMAT` | 400 | Phone number not in E.164 format |
| `PHONE_NOT_FOUND` | 404 | Phone number not found in workspace |
| `AGENT_NOT_FOUND` | 404 | Agent not found in workspace |
| `CREATE_FAILED` | 500 | Internal error creating campaign |

```json
{
  "success": false,
  "error": "Agent not found",
  "code": "AGENT_NOT_FOUND"
}
```

<Tip>
  After creating a campaign in `draft` status, you must call the [resume action](/api-reference/campaigns/manage) to start calling. Campaigns do not start automatically unless `scheduled_at` is set.
</Tip>
